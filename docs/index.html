Patterson Companies Design System

Claude Code Marketplace

The brand core, every template, and every UI kit — packaged as nine individually installable Claude Code plugins. Each ships a skill, slash commands, a specialist subagent, and a self-contained snapshot of the files it needs.

Install

Push the claude-code-marketplace/ folder to a private git repo, then add it inside Claude Code. Install only what you need — plugins are independent.

# register the marketplace
/plugin marketplace add patterson-agents/design-system

# install the foundation + whatever you need
/plugin install patterson-brand@patterson-design
/plugin install patterson-deck@patterson-design

Catalog

Every plugin is namespaced — commands run as /<plugin>:<command>, and agents are picked up automatically once installed.

What's inside each plugin

  • SkillBrand + template knowledge Claude invokes automatically, or you run directly as /plugin:skill.
  • Slash commandA scaffold workflow: copies the plugin's ds/ snapshot into your project and adapts it to your brief.
  • SubagentA specialist (deck builder, brand reviewer, storefront builder…) Claude delegates to when the task matches.
  • ds/ snapshotTokens, fonts, logos and the template's source, mirroring the design-system tree so relative paths just work.

Develop it anywhere

The repo doubles as a ready-to-prebuild workbench: open it in GitHub Codespaces or VS Code Dev Containers and claude, opencode, copilot and gh are preinstalled.

  • .devcontainer/Node 22 + gh CLI + Claude Code features; Copilot CLI and OpenCode install during prebuild.
  • .vscode/Schema-validated marketplace.json, docs live-preview, validate/serve tasks, extension picks.
  • .github/copilot-instructions.md conventions plus workflows that prebuild the image and publish the patterson-agents dev container template to GHCR.
  • dotfiles/Agent aliases + idempotent install.sh — drop into your Codespaces dotfiles repo.

Maintenance

The marketplace is generated from the design-system project. When a token or template changes at the source, re-copy the affected files into each plugin's ds/ snapshot, bump versions in plugin.json and marketplace.json, and run claude plugin validate . before pushing. Full details in the repo README.md.

Patterson logos, Proxima Nova subsets and brand imagery are proprietary — distribute this marketplace privately. Structure follows the official Claude Code plugin conventions (.claude-plugin/marketplace.json, skills/, commands/, agents/).
